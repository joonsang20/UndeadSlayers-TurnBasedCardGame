# 언데드 슬레이어즈 (Undead Slayers)

플레이 영상: https://youtu.be/_0cvnn5SmTY

Unity 버전: 2022.3.62f1

---

## 구현 기능 목록

전투 시스템
- 플레이어/적 턴 교대 진행 시스템
- 아군 카드 선택 → 행동 선택(공격/스킬) → 대상 선택 흐름
- 카드별 공격 효과 처리 (반격, 원거리 무반격, 확산 피해)
- HP 계산 및 카드 제거 처리
- 전장 빈 슬롯 발생 시 대기 카드 자동 배치
- 승리/패배/무승부 판정
- 간단한 적 AI (도발 강제 타겟, 기절 카드 스킵, 스킬 쿨다운 관리)

UI
- 전투 화면 (전장, 예비 카드, 행동 패널)
- 카드 선택 하이라이트 (선택 가능/선택됨/타겟 색상 구분)
- 플레이어/적 턴 인디케이터 이미지
- 카드 HP 실시간 표시
- 승리/패배 결과 화면 및 타이틀 복귀

카드 시스템 확장
- 4종 카드에 고유 액티브 스킬 추가
  - 보병(Infantry): 도발 — 2턴간 적의 공격을 자신에게 집중
  - 아처(Archer): 조준사격 — 단일 대상에게 1.5배 피해
  - 기마병(Cavalry): 기절 — 대상을 1턴간 행동 불능 상태로 만듦
  - 클레릭(Cleric): 힐 — 아군 단일 대상 HP 5 회복
- 스킬 쿨다운 시스템
- 도발(강제 타겟), 기절(행동 불능) 상태이상 시스템

비주얼
- DOTween 기반 전체 전투 연출
  - 근접 공격 이동 모션 (보병/성직자/기마병)
  - 원거리 화살 발사 (아처 기본 공격)
  - 조준사격 차징 + 강화 화살 연출
  - 피격 빨간 플래시 + 흔들림
  - 사망 페이드아웃 후 카드 제거
  - 신규 카드 페이드인 등장
  - 스킬별 고유 이펙트 (도발 황금 보더 펄스, 기절 보라색 유지, 힐 초록 플래시)
  - 클레릭 패시브 발동 시 아군 전체 초록 플래시 후 HP 반영
- 카드 일러스트 적용
- 타이틀 씬 및 씬 전환 페이드 연출

UX
- 행동 결과 텍스트: 각 행동 후 결과 메시지 표시
- 단계별 가이드 텍스트: 카드 선택 / 타겟 선택 안내
- 카드 툴팁: 아군 카드 터치/마우스 오버 시 공격/스킬 설명 표시

## 주요 코드 구조

```
Assets/Scripts/
├── Core/
│   ├── Data/
│   │   └── CardData.cs          # 카드 스탯/설명 ScriptableObject
│   ├── Interfaces/
│   │   ├── IAttackStrategy.cs
│   │   ├── ISkillStrategy.cs
│   │   └── IPassiveStrategy.cs
│   ├── BoardManager.cs          # 전장/예비 카드 상태 관리
│   ├── BattleResolver.cs        # 행동 실행 및 사망 처리
│   ├── CardInstance.cs          # 카드 런타임 상태 (HP, 스턴, 도발 등)
│   ├── GameManager.cs           # 전투 흐름 총괄 (코루틴 기반)
│   └── TurnManager.cs           # 턴 교대 및 패시브 발동 타이밍
├── Strategies/
│   ├── Attack/                  # 공격 전략 (Normal, Ranged, Cavalry)
│   ├── Skill/                   # 스킬 전략 (Taunt, AimShot, Stun, Heal)
│   └── Passive/                 # 패시브 전략 (HealerPassive)
├── AI/
│   └── AIController.cs          # 적 행동 결정 로직
├── Factory/
│   └── CardFactory.cs           # 카드 인스턴스 생성
└── UI/
    ├── BattleAnimator.cs        # 전투 연출 총괄 (DOTween)
    ├── BattleUI.cs              # 전투 UI 입력/표시 처리
    ├── CardView.cs              # 카드 단일 뷰 (하이라이트, 애니메이션)
    ├── TooltipUI.cs             # 카드 툴팁 표시
    ├── ResultPanel.cs           # 결과 화면
    ├── ScreenFader.cs           # 씬 전환 페이드
    └── TitleUI.cs               # 타이틀 씬
```

설계 포인트

전략 패턴 (Strategy Pattern)
공격/스킬/패시브 행동을 인터페이스로 분리하여 카드 타입별 행동을 독립적으로 구현했습니다. 새로운 카드 추가 시 기존 코드 수정 없이 전략 클래스만 추가하면 됩니다.

순수 C# 게임 로직 분리
BoardManager, BattleResolver, TurnManager, AIController는 MonoBehaviour에 의존하지 않는 순수 C# 클래스로 구현하여 게임 로직과 UI를 명확히 분리했습니다.

이벤트 기반 UI 갱신
게임 로직 레이어는 이벤트(OnHpChanged, OnTurnStarted, OnCardRefilled 등)를 발행하고 UI 레이어가 구독하는 방식으로 의존성을 단방향으로 유지했습니다.

연출 타이밍 분리
GameManager가 행동 연출 → 데미지 계산 → 피격/사망 연출 → 카드 리필 순서를 코루틴으로 제어하여 연출과 게임 로직의 타이밍을 명확히 분리했습니다.

원본 스펙 대비 설계 변경 사항

| 항목 | 원본 스펙 | 변경 내용 | 이유 |
|------|-----------|-----------|------|
| 데미지 공식 | 현재 HP만큼 피해 | 별도 ATK 스탯 사용 | 카드 성장 확장성 및 전략적 다양성 확보 |
| 기마병 확산 | 인접 카드 중 무작위 1장 | 인접 전체에 50% 피해 | 포지셔닝 전략 강화 및 일관된 게임플레이 |

## 사용한 외부 에셋 및 AI 리소스

외부 에셋
- DOTween (HOTween v2) — 전투 연출 애니메이션 라이브러리

AI 도구 활용
- Claude Code (Anthropic) — 전체 코드 설계, 구현, 디버깅에 활용
  - 아키텍처 설계 (전략 패턴, 이벤트 구조)
  - 전투 시스템, AI, UI 로직 전체 코드 작성
  - 버그 수정 및 연출 구현
- Google Gemini 이미지 생성 — 아트 리소스 제작에 활용
  - 카드 일러스트 4종 (보병, 아처, 기마병, 클레릭) 및 적군 버전 4종
  - UI 버튼 이미지, 턴 인디케이터, 화살 스프라이트, 배경 이미지
